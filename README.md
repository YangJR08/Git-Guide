# Git 指南

这是一个面向新手整理的 Git 学习仓库，重点不是只背命令，而是尽量把 Git 的核心逻辑、常用操作、以及 VS Code 连接 GitHub 的完整流程讲清楚。

如果你刚开始学 Git，希望有人用更容易跟上的方式带你从“看不懂命令”走到“能自己提交、推送、连接 GitHub”，这份内容就是按这个思路写的。

## 仓库内容

### [git教程.md](./git教程.md)
适合第一次配置 Git 和 GitHub 的同学。

主要内容包括：

- Git 安装过程中的关键选项
- VS Code 中初始化 Git 仓库
- 把本地项目连接到 GitHub
- 通过 `HTTPS` 推送代码
- 把 GitHub 远程地址从 `HTTPS` 改成 `SSH`
- 本地仓库直接通过 `SSH` 连接 GitHub 新仓库

### [git指令集.md](./git指令集.md)
适合已经开始接触 Git 命令，但总觉得“会敲命令，不知道为什么这么敲”的同学。

主要特点：

- 用更偏小白的语言解释 Git
- 不只写命令，也解释命令背后的用途
- 帮你理解工作区、暂存区、本地仓库这些概念
- 适合作为平时查命令时的参考资料

### [git教程图集](./git教程图集)
这里存放教程中配套使用的截图。

## 推荐阅读顺序

如果你是完全新手，建议按这个顺序看：

1. 先看 [git教程.md](./git教程.md)
2. 跟着把 Git、VS Code、GitHub 连接流程跑通
3. 再看 [git指令集.md](./git指令集.md)
4. 边练边理解常用命令和 Git 的核心逻辑

## 这份仓库适合谁

- 刚接触 Git 的新手
- 想学会在 VS Code 里使用 Git 的同学
- 想把本地项目上传到 GitHub 的同学
- 总是分不清 `git add`、`git commit`、`git push` 的同学
- 想弄明白 `HTTPS` 和 `SSH` 区别的同学

## 学完后你应该能做到

- 知道 Git 在管理什么
- 能在本地初始化 Git 仓库
- 能完成一次标准提交流程
- 能把本地项目推送到 GitHub
- 能把远程仓库地址从 `HTTPS` 切换成 `SSH`
- 能在 VS Code 里完成基本的 Git 日常操作

## 常用流程速看

日常最常用的一套命令通常是：

```bash
git status
git add .
git commit -m "这次改了什么"
git push
```

如果你还没把本地仓库连到 GitHub，可以先去看：

- [git教程.md](./git教程.md)

如果你想理解这些命令为什么这样用，可以再看：

- [git指令集.md](./git指令集.md)
